Day 06 —
Building a CRUD app



By : admin 27-07-2022

We explored what the Model View Controller (MVC) development pattern is and also what CRUD operations are in my previous post... Kindly click here if you missed it.

Now, let us try to apply the MVC development pattern to the creation of a realistic todo app.

Starting from the view, we want to implement an HTML form that'll be responsible for taking user input(say "a todo task") and also a button that when pushed, sends the input to the controller. The controller(charged with the control logic) then retrieves this user input, sends this data by using database sessions to the model and then manipulates the model. Now the model(responsible for the data logic), based on the data received from the controller, creates a record in our database, and returns the newly created todo item/task back to the controller. The controller then takes the newly created item, and decides how to update the "view" with it for its presentation to the user.

Another scenario where we can implement the MVC pattern is the instance of creating a bank app. The storage of transactions, profiles, balances, accounts are all part of the "Model" layer since they all are about data and the Model layer is responsible for the data/business logic. The brain and logic, also known as the "controller" is the layer responsible for running the code behind the screen and interactive elements on the app itself. The graphical user interface (GUI) represented by the actual installed app is what users see, therefore, the "view".

Now that we know the ways we can apply the MVC pattern to our development, let us start with how we accept and get user data in the context of a Flask app.

The following methods are the methods involved in getting user data in a Flask environment:

  • URL query parameters request.args.get — by listing parameters as key-value pairs at the end of a URL with a preceding question mark symbol"?"
  • Forms- request.forms.get
  • JSON- request.data
