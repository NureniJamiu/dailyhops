An Introduction to Networks

What is a Network?
A network is a conduit that connects two or more computers or other devices. It doesn't strictly have to be a connection between two computers, it can also be a computer sending data to a shared printer. And of course, most networks are more complicated than that. A situation where computers need to network with each other is in the case of a business with a considerable amount of employees, large companies that are connected to remote offices and many individual homes sharing their respective in-wall printers.

Types of Networks
There are two major types of networks, which are:
1. Local Area Network(LAN): The LAN which stands for the local area network is basically a single geographical limited network. It could be a wired or wireless network. I like to refer to these types of networks as "in-building" networks.

2. Wide Area Network(WAN): On the other hand, WAN, which stands for the wide area network is a network of two or more local area networks(LANs). A large organization may usually have the need to share data across multiple sectors which could be quiet impossible or silly to achieve using a LAN, but easily accomplished by employing the use of a WAN.

Before getting into any further detail on how a WAN operates, it'll be valuable to, first of all, explore a real life analogy of how data is sent across the network...

The Post Office Model
Imagine you want to send a post-mail to your relative living somewhere far away, the first thing you do is to write the letter in a language that your recipient understands adopting the common conventions for letter writing, the next thing is to put your letter inside an envelop that contains your address and your recipient's on it. Then you place it in the post box where the post man comes to fetch it and then transports it to the post office for further sorting(s) before finally setting it on a journey to the destination address where the recipient finally receives the letter.

Similarly, networks forward packets of data in the same manner as a post office model. We'll explore this analogy in depth later when we talk about the five layers reference model, but first, let us look at a very important concept— Network Protocols.

Network Protocols
A protocol is a formal set of written rules or procedures that computers must understand, accept and use in order to be able to communicate with one another over a network.

A real-life analogy of protocols is the human communication method. In order for a communication to be established between two or more persons, they have to communicate with a language they both/all understand. A protocol is more than merely a language, but also the rules of etiquette frequently encountered among spoken languages. It is typical for someone who starts a conversation with a "hi" message to wait for the receiver's response. He/She should also know when to respond and not just interrupt the other in the middle of an exchange. Network protocols similarly set the rules such as when computers are able to transmit data, which computer the data is destined for and what to do if the data is not received.

To grasp a deeper understanding about what protocols are and how they operate, let us examine the first fundamental types of network models.

1. OSI Model: The Open System Interconnection model (OSI) provides guidelines on how computers communicate over a network. It is made up of 7 layers. The OSI model defines how data should be passed from one layer to the next. The sending computer sends data down the network stack/suite, while the receiving computer sends the received data up the stack/suite. The layers of the model are:
  • Application layer (Layer 7)
  • Presentation layer (Layer 6)
  • Session layer (Layer 5)
  • Transport layer (Layer 4)
  • Network layer (Layer 3)
  • Data-link layer (Layer 2)
  • Physical layer (Layer 1)
The OSI model identifies the steps and functions that must be completed at each layer when computers communicate over a network. It is important to note that this model does not define specific details on how to actually make this happen.

2. TCP/IP Model: The transmission control protocol/internet protocol (TCP/IP) model is a 4 layer model that governs internet communications. This model is developed by the department of defense and internet engineering task force. This model defines specific protocols at each layer of the stack such as the TCP and IP which are two of the internet core protocols. The model consists of the:
  • Application layer (Layer 4)
  • Transport layer (Layer 3)
  • Internet layer (Layer 2)
  • Network Access layer (Layer 1)
Note that in an OSI model, the application, presentation and session layers all make up the single application layer in a TCP/IP model.

We'll learn more about these layers when we get to the five layers reference model
