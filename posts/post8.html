Day 08 —
The Five-Layer Model



By : admin 15-08-22

The Five-Layer model apparently consists of five layers:
  • Application layer (Layer 5)
  • Transport layer (Layer 4)
  • Network layer (Layer 3)
  • Data link layer (Layer 2)
  • Physical layer (Layer 1)
Let us look at what happens in each of these layers.

1. Application layer: The application layer contains some protocols that usually specify how data should be encoded, compressed, or encrypted and also how sessions should be managed before passing the data to the layer below. These protocols include: Hypertext transfer protocol(HTTP), HTTPS, File Transfer Protocol(FTP), Domani Name System(DNS), Simple Mail Transfer Protocol(SMTP) and some many more...

Using our postal mail analogy, our data at this point is simply a letter which has been written in a language that the recipient understands and the traditional conventions for letter writing has been adopted.

2. Transport Layer: This layer specifies which application layer protocol should be used to process the data once it arrives on the receiving computer. On this layer, each application protocol is assigned a unique numerical identifier, called a port number. There are various port numbers that particularly assigned to specific application layer protocols. Some of the popular ones are: port 80 which is assigned to the HTTP protocol, port 53, assigned to DNS, port 25 to SMTP and so on... These specific port numbers are included by the transport layer in the header it adds to the data. This addressing information contained in the header or sometimes, the footer of the data is examined by the transport layer on the receiving computer as well. The receiving computer, in order to perform this examination, strips the header of the data and determines the suitable application layer protocol the data should be sent to for further processing.This process of adding header and footers/trailers to the data layer by layer is known as encapsulation, while the process of removing/stripping off the header/footer at each layer of the stack is known as de-encapsulation.

Beyond identifying the destination and source port numbers, the transport layer has a second responsibility— the choice of reliability, which brings us to the two main protocols functioning in the transport layer: the UDP, and TCP.

UDP stands for User Datagram Protocol. It is a commonly referred to as a best effort delivery service due to the fact that it is fast and time-sensitive. It doesn't notify the sender of a successful delivery message, no error checking and no recovery procedure. UDP is used in online games, streaming videos, Voice over IP(VoIP), and also DNS where reliability of a data sent is not much expected.

On the other hand, TCP which is the Transmission Control Protocol is the entire opposite of UDP. It is highly concerned with reliability. It ensures that data is delivered safely to the intended receiver, checks for errors and if for any reason, the data transfer could not be completed, it resends the data until it is successfully delivered. This processes of error checking, re-transmitting of data and delivery notification makes TCP a relatively slow protocol when compared to UDP.

How does TCP work?
TCP accepts the data sent from the application layer protocol, and slices the data into smaller pieces called "Segments". This is to ensure a steady flow of data between computers regardless of the difference in connection bandwidth. TCP assigns a sequence number to each segment which can be used to put the segments back together correctly on the receiving end.

TCP is equivalent to sending our letter by a registered mail with a return receipt which the post office returns to you upon successful delivery of your letter, while UDP is just like you putting a stamp on your mail and just placing it in the post office box without knowing if the letter got delivered to it's destination at all.

Note: It is valuable to keep note of the terms that the actual data is called in each layer of the model. Here in the transport layer, it is called " Segments".

3. Network Layer: On the passing of the segments from the transport layer down to the network layer, the network layer takes a segment and adds it's own header to make it a "packet". This header contains a destination and source IP address. An IP address, also known as a Logical address is a unique address that identifies a host on a network.

The network layer also adds, as part of it's header, the number identifying the upper transport layer protocol. Each transport layer(Layer 4) is assigned a unique identifier also know as the IP protocol number. The number for UDP is 17 while that of TCP is 6. The receiving computer on the other end, examined this IP protocol number and determines with this information contained in the header, which upper layer protocol should process the data next.

4. Data Link Layer: This layer also adds its own header to the data sent down from the layer just above it. Then the "packet" becomes a "frame". This header includes another address commonly referred to as the Layer 2 address, physical address, hardware address or the Media Access Control(MAC) address. This address can be compared to the addressing on the post-office's sorting containers. Typically, the header also includes an indication as to which layer 3 protocol is in the data portion of the frame.

Layer 2 also performs its own data integrity check so that the corrupted or damaged frames are discarded as soon as possible. The check is also done by adding a "checksum" in the trailer at the end of the frame. A checksum is a result of a mathematical calculation run by both the sending and receiving computers and inserted by the sending computer into the frame's trailer. Layer 2 finally converts this frame into a format that the physical layer can understand and that is the bit and bytes (ones and zeros) of the digital communications.

5. Physical layer: This layer finally converts this bits of data into electrical signals and sends them across the physical medium which could be a telephone wire, fibre-optic cable, or even a wireless environment). This layer specifications define the characteristics such as cabling specifications, voltage levels, physical data rates, maximum transmission distances, and physical connectors.

The receiving computer receives these signals, and starts the process of de-encapsulation. It removes the headers from each layer and repeats the process from the ground layer up to the 7th layer where the actual data is finally delivered to its intended destination.
